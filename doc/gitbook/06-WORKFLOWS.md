# Fluxos de Trabalho

## Objetivo

Registrar como o time transforma demanda em entrega.

## Fluxo minimo recomendado

1. Receber tarefa
2. Confirmar entendimento
3. Criar branch
4. Implementar
5. Testar localmente
6. Abrir revisao
7. Validar
8. Integrar
9. Atualizar documentacao, se necessario

## Tipos de fluxo

### Correcao de bug

- coletar reproducao;
- validar causa;
- corrigir com escopo pequeno;
- registrar risco de regressao.

### Feature

- alinhar criterio de aceite;
- quebrar em etapas menores;
- documentar impacto em UX, build e teste.

### Release

- consolidar itens;
- validar pendencias;
- gerar artefatos;
- registrar versao entregue.

## Relacao com a documentacao

Atualize a doc sempre que houver:

- novo fluxo operacional;
- mudanca estrutural relevante;
- novo risco conhecido;
- nova dependencia de ambiente.
